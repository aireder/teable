# Grid Table Kanban Demo 启动说明

## 快速启动

### 方法一：使用启动脚本（推荐）
```bash
cd demo
./start-demo.sh
```

### 方法二：手动启动
```bash
cd demo
npm install
npm run dev
```

## 访问演示

启动成功后，在浏览器中访问：
- **本地地址**: http://localhost:3000
- **网络地址**: http://[你的IP]:3000

## 演示功能

### 🚀 Grid 表格演示
- 高性能 Canvas 渲染
- 支持 100 到 100,000 行数据
- 多种单元格类型（文本、布尔、评分、选择器）
- 实时编辑和验证
- 虚拟滚动性能展示

### 📋 Kanban 看板演示
- 7 列完整工作流程
- 任务生命周期管理
- 子任务和验收标准
- 严格的质量门控
- 拖拽任务移动

## 系统要求

- Node.js 16+ 
- 现代浏览器（Chrome 90+, Firefox 88+, Safari 14+）
- 至少 4GB 内存（用于大数据量测试）

## 故障排除

### 端口被占用
如果 3000 端口被占用，可以修改 `vite.config.ts` 中的端口配置：
```typescript
server: {
  port: 3001, // 改为其他端口
  host: true
}
```

### 依赖安装失败
```bash
# 清理缓存重新安装
rm -rf node_modules package-lock.json
npm install
```

### 构建失败
```bash
# 检查 TypeScript 类型
npm run typecheck
```

## 开发模式

在开发模式下，代码会自动热重载，修改源码后浏览器会自动刷新。

## 生产构建

```bash
npm run build
```

构建完成后，静态文件将输出到 `dist` 目录。

**提示**: 首次启动可能需要几分钟时间来安装依赖，请耐心等待。
